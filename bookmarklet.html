Key Metrics Bookmarklet

Drag the link below to your bookmarks bar. When on a page with the creator selector, Key metrics panel, and Videos table, click it to console.log a single object and (if configured) POST it to a Google Sheet:

{
  creator,
  scrapedAt,                 // ISO timestamp of the scrape
  dateRange: { start, end }, // selected Month date-range picker values
  metrics: [ { name, value, trend } ],
  videos:  [ { "Video ID", "Name", "Link", "Posted", "Duration",
               "Affiliate GMV", "Items sold", "Est. commissions",
               "Direct GMV", "RPM", "Views", "Completion rate", "Details" } ]
}

Inactive video rows are skipped. To enable the Google Sheet sync, see Google Sheet setup.

Google Sheet setup

The bookmarklet can POST each scrape directly into a Google Sheet via a Google Apps Script Web App. This avoids OAuth in the browser and writes into two tabs:

  • Metrics — one row per metric per scrape (append-only time series, with date_start / date_end from the selected month range).
  • Videos — one row per video, upserted by Video ID (re-runs update existing rows; new videos append).
  1. Create a new Google Sheet. (Tabs Metrics and Videos will be created automatically on first write, or you can create them yourself.)
  2. Open Extensions → Apps Script and replace Code.gs with the contents of apps-script/Code.gs in this repo. Save.
  3. Open Project Settings → Script properties and add property TOKEN with a random shared secret (e.g. openssl rand -hex 16).
  4. Click Deploy → New deployment. Choose type Web app. Set Execute as: Me and Who has access: Anyone. Copy the /exec URL.
  5. In the bookmarklet source above, paste the /exec URL into ENDPOINT and the same shared secret into TOKEN. Re-URL-encode the javascript: source, update the <a class="bm" href="…">, and re-drag the bookmark. (If ENDPOINT is left as the placeholder, the bookmarklet silently skips the POST and continues to console.log only.)

Verify end-to-end by clicking the bookmarklet on a TikTok Shop analytics page: DevTools → Network should show a POST to the /exec URL returning {ok:true, metricsWritten, videosUpserted}, and the Sheet should gain rows. An invalid token returns {ok:false, error:"unauthorized"} and leaves the sheet unchanged.

Graylog setup

The bookmarklet also fires a second POST to a local Graylog server via its GELF HTTP input. Full install steps (Windows/WSL2, native MongoDB + OpenSearch + Graylog) are in graylog-local-setup.md.

  1. Follow graylog-local-setup.md to get Graylog running at http://localhost:9000 and a GELF HTTP input listening on port 12202 with CORS enabled.
  2. In the bookmarklet source above, set GRAYLOG_ENDPOINT to http://localhost:12202/gelf (or http://<WSL-IP>:12202/gelf if localhost doesn't reach WSL). Leave it as '' to disable Graylog posting.
  3. Re-URL-encode the source and update the bookmarklet link.

The Graylog message is sent as GELF 1.1 with host: "tiktok-bookmarklet" and these custom fields: _creator, _scrapedAt, _date_start, _date_end, _metrics_count, _videos_count, _metrics_json (stringified array), _videos_json (stringified array). The request uses fetch(..., { mode: 'no-cors', headers: { 'Content-Type': 'text/plain;charset=utf-8' } }) so it is a CORS "simple request" (no preflight) and the bookmarklet cannot read the response body — check DevTools → Network for a 202, or search host:tiktok-bookmarklet in the Graylog UI.
